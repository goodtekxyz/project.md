# Demo Project

## One-liner

A fictional project used to demonstrate the project.md format.

## Mission

Show how humans and AI agents can manage a product project through structured Markdown files.

## Current Status

Discovery

## Current Focus

Define the core problem and validate whether the project is useful.

## Key Ideas

- Conversations should update project knowledge.
- `PROJECT.md` should represent the current state.
- `HISTORY.md` should preserve important reasoning over time.

## Decisions

- Start with only `PROJECT.md` and `HISTORY.md`.
- Add new files only when the project becomes complex enough.

## Open Questions

- What is the smallest useful project structure?
- How should AI agents decide when to update history?
